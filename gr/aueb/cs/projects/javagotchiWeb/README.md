# 🐾 Javagotchi Web: Το Απόλυτο Καταφύγιο Εικονικών Κατοικιδίων

Το **Javagotchi Web** είναι μια ολοκληρωμένη Web εφαρμογή εικονικών κατοικιδίων, η οποία αναπτύχθηκε με **Java Spring Boot**. Αποτελεί την εξέλιξη ενός αρχικού project κονσόλας σε μια σύγχρονη πλατφόρμα που συνδυάζει την αντικειμενοστραφή σχεδίαση με τις τεχνολογίες διαδικτύου.

---

## 🎯 Επισκόπηση Έργου
Το project δημιουργήθηκε με σκοπό να προσφέρει μια διασκεδαστική εμπειρία διαχείρισης κατοικιδίων (στυλ Tamagotchi), επιτρέποντας σε πολλούς χρήστες να διατηρούν τα δικά τους καταφύγια ταυτόχρονα. Κάθε χρήστης μπορεί να υιοθετεί, να ταΐζει και να παίζει με τα κατοικίδιά του, προσπαθώντας να τα κρατήσει ζωντανά καθώς ο χρόνος περνάει.

---

## 🛠️ Τεχνική Ανάλυση

Η εφαρμογή έχει δομηθεί με βάση τις βέλτιστες πρακτικές του Αντικειμενοστραφούς Προγραμματισμού (OOP) και της αρχιτεκτονικής Web:

1.  **Κληρονομικότητα & Πολυμορφισμός:**
    * Η βασική κλάση `Pet` ορίζει τις κοινές ιδιότητες (name, hunger, happiness).
    * Οι κλάσεις `DragonPet` και `SlimePet` επεκτείνουν την `Pet`, επαναορίζοντας (override) τη μέθοδο `tick()` για διαφορετικούς ρυθμούς μείωσης στατιστικών και τη μέθοδο `checkStatus()` για εξειδικευμένα μηνύματα.
2.  **Singleton Design Pattern:**
    * Η κλάση `ShelterManager` υλοποιεί το πρότυπο Singleton, διασφαλίζοντας ότι υπάρχει μόνο ένας κεντρικός διαχειριστής δεδομένων σε όλη τη διάρκεια λειτουργίας του server.
3.  **Πολυνηματικότητα (Multithreading):**
    * Χρήση του `ScheduledExecutorService` για τη δημιουργία ενός background "ρολογιού" (Global Timer). Αυτό επιτρέπει τη σταδιακή μείωση των στατιστικών των κατοικιδίων όλων των χρηστών ανεξάρτητα από τη δραστηριότητά τους στον browser.
4.  **Αρχιτεκτονική MVC:**
    * **Model:** Οι κλάσεις των Pets και ο Manager.
    * **View:** Δυναμικά templates HTML5 με τη χρήση της μηχανής **Thymeleaf**.
    * **Controller:** Ο `JavagotchiController` διαχειρίζεται τα requests, το routing και τη ροή της εφαρμογής.
5.  **Ασφαλής Διαχείριση Δεδομένων:**
    * Χρήση `ConcurrentHashMap` για την αποθήκευση των καταφυγίων, διασφαλίζοντας την ακεραιότητα των δεδομένων σε περιβάλλον πολλαπλών ταυτόχρονων χρηστών.

## 📂 Δομή Project
* `src/main/java/com/example/javagotchi/`: Ο πηγαίος κώδικας Java.
* `src/main/resources/templates/`: Οι ιστοσελίδες HTML (Thymeleaf).
* `src/main/resources/static/css/`: Το αρχείο `style.css` για τον σχεδιασμό.

## 🔧 Εγκατάσταση & Εκτέλεση
1.  Κάντε clone το repository.
2.  Ανοίξτε το project στο IntelliJ IDEA ή στο Eclipse ως **Maven Project**.
3.  Εκτελέστε την κλάση `JavagotchiApplication.java`.
4.  Επισκεφθείτε τη διεύθυνση `http://localhost:8080` στον browser σας.
> **Σημείωση:** Το default port είναι το `8080`. Αν θέλετε να το αλλάξετε, μπορείτε να το κάνετε μέσω του αρχείου `src/main/resources/application.properties` αλλάζοντας το `server.port = <port_number>`.

## 🌐 Οδηγός Εγκατάστασης (Local ή Linux Server)

Ένα έτοιμο, εκτελέσιμο αρχείο .jar (**JavagotchiWeb-1.0-SNAPSHOT.jar**) παρέχεται ήδη στον κεντρικό φάκελο του project για άμεση εκτέλεση.

### 1. Άμεση Εκτέλεση (Quick Start)
Αν έχετε εγκατεστημένη τη Java 17+ (δοκιμάστηκε σε Java 25), μπορείτε να τρέξετε την εφαρμογή κατευθείαν ανοίγοντας ένα τερματικό στον φάκελο του project και εκτελώντας:

java -jar JavagotchiWeb-1.0-SNAPSHOT.jar

Η εφαρμογή θα ξεκινήσει τοπικά στο http://localhost:8080.

### 2. Ανέβασμα σε Linux Server
Αν επιθυμείτε να φιλοξενήσετε την εφαρμογή σε απομακρυσμένο server:

1. **Μεταφορά:** Χρησιμοποιήστε FileZilla για να στείλετε το έτοιμο .jar αρχείο στον server:
   scp JavagotchiWeb-1.0-SNAPSHOT.jar username@server_ip:/home/username/

2. **Εγκατάσταση Java:** Συνδεθείτε στον server μέσω SSH και εγκαταστήστε την Java 17:
   sudo apt update && sudo apt install openjdk-17-jre -y

3. **Εκτέλεση στο παρασκήνιο (Systemd):**
   Συστήνεται η δημιουργία ενός service στο /etc/systemd/system/javagotchi.service για να τρέχει η εφαρμογή συνεχώς:
    - **ExecStart:** Η πλήρης διαδρομή προς την Java και το αρχείο .jar.
    - **Restart:** always (για αυτόματη επανεκκίνηση σε περίπτωση σφάλματος).

### 3. Δημιουργία νέου JAR από τον πηγαίο κώδικα (Σε περιβάλλον Linux)
Αν κάνετε αλλαγές στον κώδικα και θέλετε να παράξετε ένα νέο `.jar` αρχείο απευθείας από το τερματικό:

1. **Εγκατάσταση Προαπαιτούμενων:** Βεβαιωθείτε ότι έχετε εγκατεστημένα το **JDK** και το **Maven**. Σε διανομές Ubuntu/Debian εκτελέστε:
   ```bash
   sudo apt update
   sudo apt install openjdk-17-jdk maven -y
   ```

2. **Πλοήγηση στον φάκελο:** Μεταβείτε στον κεντρικό φάκελο του project (εκεί που βρίσκεται το `pom.xml`):
   ```bash
   cd /διαδρομή/προς/τον/φάκελο/javagotchi
   ```

3. **Δημιουργία του JAR:** Εκτελέστε την παρακάτω εντολή για να κατεβούν οι βιβλιοθήκες και να γίνει το build:
   ```bash
   mvn clean package
   ```

4. **Εκτέλεση:** Μόλις δείτε το μήνυμα `BUILD SUCCESS`, το νέο εκτελέσιμο αρχείο θα έχει δημιουργηθεί μέσα στον φάκελο `target/`. Μπορείτε να το τρέξετε με:
   ```bash
   java -jar target/JavagotchiWeb-1.0-SNAPSHOT.jar
   ```
